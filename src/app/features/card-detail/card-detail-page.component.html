Main Character Behavior Per Stage

The character is always the SAME chibi person. Their posture tells the emotional story.

STORM Collapsed, curled, hiding, fists clenched, looking down or away. Worn dark oversized clothing — hiding in fabric.
FOG Reaching out blindly, standing still in confusion, half-turned, uncertain gaze. Muted tones, layered, still modest.
CLEARING Lifting head, first eye contact with viewer, tentative step forward, hand on heart. Lighter fabrics, soft pastels.
AURA Standing tall, arms beginning to open, chin raised, quiet confidence. Elegant jewel tones (purple, teal), flowing silhouettes.
RADIANCE Dynamic pose, arms spread wide, genuine smile, radiating energy. Luxurious gold accents, confident fashion.
LEGEND Sovereign posture, one hand blessing/inviting, serene and powerful. Mythic ornate gown with celestial details, crown or halo.
Background Figures (2-4 per image)

Derived from the user's fear — NOT hardcoded. The AI reads the fear and decides how bystanders would behave if that fear were true.

STORM Figures EMBODY the fear — turning backs, laughing, succeeding while character fails, looking through them as if invisible.
FOG Figures blurred, distant, unreachable — character reaches out but no one responds.
CLEARING ONE figure pauses, looks back, extends a hand — the first crack in the loneliness.
AURA 2-3 figures drawn closer, watching with genuine curiosity or respect, mirroring the character's posture.
RADIANCE Figures celebrating WITH the character — embracing, cheering, sharing the light.
LEGEND Figures beginning their OWN journeys inspired by the character — looking up in awe, telling stories, following the path.
Visual Mood Per Stage
STORM Dark stormy sky, rain, cold blue-purple tones, cracked ground, dim lighting.
FOG Silver-grey mist, diffused light, low visibility, dreamlike and disorienting.
CLEARING Golden hour breakthrough, sun rays piercing clouds, warm amber light, first green growth.
AURA Twilight purple sky, ethereal energy, constellation patterns, mystical floating particles.
RADIANCE Full golden sunlight, warm sparks, lens flares, everything glowing.
LEGEND Cherry blossoms falling, celestial twilight, both stars and dawn visible, timeless and mythic.
Quality Check — Does Each Image...
☐ Reflect the user's specific fear (not generic sadness)?
☐ Show the SAME character across all 6 stages?
☐ Have 2-4 background figures whose behavior matches the stage?
☐ Show outfit progression from worn → regal?
☐ Have the correct mood lighting and atmosphere?
☐ End with tarot card composition and gold border?
☐ Contain NO text, letters, or watermarks?
Prompt Writing Rules

These rules are enforced in buildMetaPrompt() — the AI must follow them for every image prompt.

✓ Write ONLY visual descriptions — no abstract emotions ("cautious optimism")
✓ Describe PHYSICAL expression: "eyes wide, slight smile, chin raised"
✓ Specify exact COLOR of every element (hair, outfit, sky, ground)
✓ Background figures: exact pose + position ("turned 45° away, 3ft left")
✓ Every prompt self-contained — image AI has no memory between stages
✓ Never reference "Storm version" — describe visually: "smaller grey silhouette in torn hoodie"

Image Provider Architecture

Design Pattern

Strategy Pattern — ImageProvider interface with swappable implementations. The active provider is selected by app.image-provider in application.yml.

Architecture
CardImageGenerationService
↓ imageProvider.generate(prompt)
ImageProvider (interface)
↓ selected by app.image-provider config
Gemini ModelsLab + Future
Available Providers
GEMINI Google AI Imagen. Supports chibi reference image for character consistency. Config: app.gemini.api-key, app.gemini.model. Set: app.image-provider=gemini
MODELSLAB Stable Diffusion via ModelsLab API. Best anime quality (animagine-xl). Auto-appends anime style + negative prompt. Handles async polling. Config: app.modelslab.api-key, app.modelslab.model-id, width, height. Set: app.image-provider=modelslab
Adding a New Provider
1. Create NewProviderImageProvider.java implementing ImageProvider
2. Annotate with @Component("providername")
3. Implement generate(prompt, chibiUrl) → return base64 PNG
4. Implement providerName() → return display name
5. Add config under app.providername.* in application.yml
6. Set app.image-provider=providername to activate
✓ No changes needed in CardImageGenerationService or any other class
